# 1 引言
## 1.1 产品定义
### 1.1.1 产品目标
本产品为中国海洋大学校园二手物品交易平台，采用安卓app的形式。目的是建立一个方便安全的平台，用以解决校园二手物品信息不对称的问题。我们会通过在学校论坛及贴吧发帖、通过微信公众号撰写推送文章、在QQ空间和微信朋友圈进行宣传等措施进行推广，最终形成覆盖海大全体师生的二手物品交易平台。

### 1.1.2 行业背景
每当开学季大量涌入的新生他们需要购置生活用品，学习材料，学习用品等，形成了大量购物的需求狂潮。而毕业季老生离校带不走的生活用品和学习材料等难以找到合适的平台和卖家出售，形成了大量的浪费，令人可惜。平日需求里，由于大学生喜好买东西，闲置下来的东西，累积下来是个负担，需要置换来各取所需。大部分体育课需要自行购买器材或装备，但这些器材只用一学期就被闲置，有需要的同学又难以找到便宜的二手器材。根据这些现象可以看出，目前海大急需一个专门的二手物品交易平台，来解决想买东西找不到卖家，想卖又没地发布消息的问题。

### 1.1.3 产品适用范围
本产品用户为海大全体师生，用于满足他们买卖二手物品的需求。

# 2 任务概述
## 2.1 产品目标分析
目前还没有基于校园范围的专门的二手物品交易平台，这是一个全新的产品。


## 2.2 用户特点分析
用户为海大师生，卖方想以理想的价格卖掉闲置的物品，买方想找到质量可靠的的所需物品。本平台便可使卖方找到发布信息的地方，并可标上想出售的价格，买方可以非常方便的搜素信息，找到需要的物品，并获得卖方的联系方式，两方可以线下见面交易，安全方便。平台用户皆为实名认证，并有管理员对要卖出物品的合法性合理性进行审核，保证了信息的真实可靠。


## 2.3 假定约束分析
### 2.3.1 直接约束
本产品为安卓app,拥有安卓手机的海大师生才可正常使用。

### 2.3.2 选择约束
软件采用java语言编写，使用的工具有IDEA Community Edition 2019.2.3、JDK 1.8、Android SDK API 29。需在本学期的5-8周(2019/10/14 -- 2019/11/10)完成α迭代，做出UI界面美观，系统运行不出现崩溃问题的app。
## 2.4 功能分析
卖家：发布消息（名称，照片，预期价格，联系方式，卖家姓名，卖家地址，描述，类别）、修改消息（修改信息，设为已卖出）、删除消息、查询自己的记录
买家：查看、搜索（地址，名称，类别）
管理员：审核消息（通过，不通过）
用户：登录（用户，管理员），注销

# 3 外部接口需求
本软件需要对接中国海洋大学信息门户实名认证接口，以保证用户信息的真实性。

## 3.1 用户界面
系统界面设计应体现人性化，界面清晰简捷，操作简单流畅，提高管理效果。
普通用户界面分为浏览物品界面和“我的”界面，浏览物品界面可以查看按随机顺序排列而成的所有待卖物品和搜索想要的物品，点击物品可以查看该物品的详细信息，“我的”界面可以发布想卖的物品信息和查看自己历史上所卖的物品。
管理员界面上将卖家所发布的信息逐条显示，点开查看具体信息后可以设置该信息是否予以发布。
## 3.2 硬件设备

# 5 非功能需求
## 5.1 性能需求
1.数据精确度 
查询信息时应保证查全率，所有相应域包含查询关键字的记录都应该查到。
查询信息应保证查准率，查到的记录应与给定的查询条件完全匹配。 
2.时间特性
本网站具体时间特性要求要根据网速来决定。我们将最大限度的减少系统响应时间，最小化更新处理时间和数据转换时间
3.系统容量需求 
注册用户：3500以上 
在线用户：1500以上 
并发数：500以上
4.适应性 
满足用户的使用需求。
## 5.2 安全需求
### 5.2.1 系统安全需求
1.设置权限，对于用户，只分配其所需权限。让其只能做自己能做的操作。
2.数据库和服务器应具有一定规模，避免恶意的大量数据攻击造成软件瘫痪。
### 5.2.2 数据安全需求
1.用户的隐私信息需要得到保护，注册时我们会将用户的密码哈希后储存，登录时仍是将所输密码哈希后与所存的数据对比来确认是否正确。这样使得开发和管理人员也不知道用户的密码，保障了用户的隐私。用户的各种信息未经用户本人许可不会提供给他人。
2.对于数据库中的数据，一定要进行实时的备份操作。每当数据库中发生一些修改操作时，就进行备份操作，这样的话，再后来的数据丢失时，就可以进行及时的恢复，从而避免了数据丢失造成的损失。
## 5.3 其它软件质量属性
1.易使用性
软件界面设计友好简洁，操作简单，用户不需经过培训便可以轻松使用。
2.可靠性 
对于编写好的软件，会进行大量的测试，不断地查找里面出现的bug，并及时的对其进行修改，尽可能的减少bug的数量。随着用户量的增加,我们会及时的更新我们的服务器和数据库，从而保证软件的可靠性。避免用户量太大，而造成服务器瘫痪。
3.完整性 
要求能在发生意外的情况下，保证不丢失数据。
