# 1 引言
## 1.1 产品定义
### 1.1.1 产品目标
本产品为中国海洋大学校园二手物品交易平台，采用安卓app的形式。目的是建立一个方便安全的平台，用以解决校园二手物品信息不对称的问题。我们首先会通过在学校论坛及贴吧发帖、通过微信公众号撰写推送文章、在QQ空间和微信朋友圈进行宣传等措施进行推广，最终形成覆盖海大全体师生的二手物品交易平台。这是一个非盈利的平台，由在校学生管理和维护系统，毕业后在下一届中寻找自愿管理的人。云服务器主要由广告收入进行维持，也欢迎志同道合的同学适当捐款维持平台运行，当然我们也希望能得到学校的支持成为一个官方平台。

### 1.1.2 行业背景
高校的二手交易市场是一个大市场，每年都有毕业生要毕业，每年都有新生来报到，毕业生们不可能将几年来留存下的东西都带走，有些物品已成为累赘，卖掉反而还能赚些路费。学生们的消费水平本来也不高，都希望能买到物美价廉的商品，这个时候，网上二手交易市场的作用就发挥出来了。卖主可以在网上登记自己要处理的物品以及售卖价格，联系方式，买主则可以在网上搜索自己所需要的。当然，网上二手交易市场不但能处理毕业生的多余物品，平时学生们有任何闲置的东西或有任何需求都可以免费在网上二手交易市场上得到处理，并且不限时间也不限地点。但目前海大并没有一个专门的二手物品交易平台，导致二手物品交易信息严重不对称。

### 1.1.3 产品适用范围
本产品用户为海大全体师生，用于满足他们买卖二手物品的需求。

# 2 任务概述
## 2.1 产品目标分析
二手交易行业已有闲鱼、转转等软件，但目前还没有基于校园范围的专门的二手物品交易平台，这是一个较新的产品。目标是做出一个简单易用的校园二手物品交易平台，用户覆盖海大全体师生，提供买卖二手物品所需的全部功能，解决他们买卖不方便的问题。


## 2.2 用户特点分析
用户为海大师生，经常会有买卖二手物品的需求，卖方想以理想的价格卖掉闲置的物品，买方想找到质量可靠的的所需物品。本平台便可使卖方找到发布信息的地方，并可标上想出售的价格，买方可以非常方便的搜素信息，找到需要的物品，并获得卖方的联系方式，两方可以线下见面交易，安全方便。平台用户皆为实名认证，并有管理员对要卖出物品的合法性合理性进行审核，保证了信息的真实可靠。


## 2.3 假定约束分析
软件采用java语言编写，使用的工具有IDEA Community Edition 2019.2.3、JDK 1.8、Android SDK API 29。
需在本学期的5-8周(2019/10/14 -- 2019/11/10)完成α迭代，做出UI界面美观，系统运行不出现崩溃问题的app。
软件运行需要连接云服务器和数据库。
## 2.4 功能分析
卖家：发布消息（名称，照片，预期价格，联系方式，卖家姓名，卖家地址，描述，类别）、修改消息（修改信息，设为已卖出）、删除消息、查询自己的记录

买家：查看、搜索（地址，名称，类别）

管理员：审核消息（通过，不通过）

用户：登录（用户，管理员），注销

# 3 外部接口需求

## 3.1 用户界面
系统界面设计应体现人性化，界面清晰简捷，操作简单流畅，提高管理效果。进入软件后会出现登录界面，用户需登录后才可使用。

普通用户界面分为浏览物品界面和“我的”界面，浏览物品界面可以查看按随机顺序排列而成的所有待卖物品和搜索想要的物品，点击物品可以查看该物品的详细信息，“我的”界面可以发布想卖的物品信息和查看自己历史上所卖的物品。

管理员界面上将卖家所发布的信息逐条显示，点开查看具体信息后可以设置该信息是否予以发布。
## 3.2 硬件设备
运行该平台所需要的硬件设备：一台接入网络的安卓手机

## 3.3 软件接口
（1）使用云服务器数据库存储数据信息。

（2）本软件需要对接中国海洋大学信息门户实名认证接口，以保证用户信息的真实性。

（3）运行于安卓系统之上。

## 3.4 通信接口
项目中用到的网络通信协议如下： 
（1）TCP/IP网络协议。
（2）通讯采用UDP协议。 
（3）HTTP协议。

# 4 功能描述
# 4.1 软件结构
数据访问层:对数据的操作层，提供对文件数据的增加、删除、修改、查询操作

业务逻辑层:主要是针对具体的问题的操作、对数据层的操作、对数据业务逻辑处理。

表示层:与用户交互的界面。用于为用户展示信息、接收用户输入的数据和显示处理后用户需要的数据。

业务实体:贯穿于三层，在三层之间传递数据，连接三层

## 4.2 登录功能
软件需要先登录才能使用，对接中国海洋大学信息门户数据，采用信息门户的账号密码进行登录，保证用户为真实的海大师生。

## 4.3 管理员功能
防止部分用户发送不良信息，此软件设有系统管理员，拥有一个独立的账号，登录后界面会逐条显示所有的用户发布但未被审核地位物品信息，管理员点击信息可查看该信息的详细内容，确定信息合法合理可以发布后该物品信息才可被普通用户看到。
## 4.4 普通用户功能
所有用户都可以同时作为卖家和买家参与卖和买。卖家可以发布物品信息，包括名称、照片、预期价格、联系方式、卖家姓名、卖家地址、描述、类别，如果发现其他状况可以修改或删除上述所有的信息。物品卖出后需卖家自己点击“已卖出”来更改物品状态，使其不再会被买家搜索到。卖家在“我的”界面可查询自己的历史出售记录，包括已卖出的、正在发布的和正在审核的物品。

买家可以在主页上浏览所有物品信息，并可在上方搜素框搜索自己需要的物品，搜索框有选择按钮，可以使用户选择某一类别进行搜索。搜索后会展示该类型所有物品，点击物品可查看物品详细信息和卖家联系信息，买家自行联系卖家进行交易。

# 5 非功能需求
## 5.1 性能需求
1.数据精确度 

查询信息时应保证查全率，所有相应域包含查询关键字的记录都应该查到。

查询信息应保证查准率，查到的记录应与给定的查询条件完全匹配。 

2.时间特性

在硬件和网络条件满足的前提下，所有日常性操作事务的平均响应时间应小于10秒，最长响应时间应小于30秒；对于查询性事务的平均响应时间应小于60秒，最长响应时间应小于180秒。

3.系统容量需求 

注册用户：3500以上 

在线用户：1500以上 

并发数：500以上

4.适应性 

满足用户的使用需求。
## 5.2 安全需求
### 5.2.1 系统安全需求
1.设置用户权限，对用户只分配其所需权限，让其只能做自己能做的操作。

2.数据库和服务器应具有一定规模，避免恶意的大量数据攻击造成软件瘫痪。
### 5.2.2 数据安全需求
1.鉴于保护用户隐私的问题，注册时不需用户输入姓名，账户的用户名为用户本人学号。对账户密码我们会哈希后储存，登录时仍是将所输密码哈希后与所存的数据对比来确认是否正确。这样使得开发和管理人员也不知道用户的密码，保障了用户的隐私。用户的各种信息未经用户本人许可不会提供给他人。

2.对于数据库中的数据，一定要进行实时的备份操作。每当数据库中发生一些修改操作时，就进行备份操作，这样的话，再后来的数据丢失时，就可以进行及时的恢复，从而避免了数据丢失造成的损失。
## 5.3 其它软件质量属性
1.易使用性

软件界面设计友好简洁，操作简单，用户不需经过培训便可以轻松使用。

2.可靠性 

对于编写好的软件，会进行大量的测试，不断地查找里面出现的bug，并及时的对其进行修改，尽可能的减少bug的数量。随着用户量的增加,我们会及时的更新我们的服务器和数据库，从而保证软件的可靠性。避免用户量太大，而造成服务器瘫痪。

3.完整性 

要求能在发生意外的情况下，保证不丢失数据。
